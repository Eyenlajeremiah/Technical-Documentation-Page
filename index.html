Introduction

React is a JavaScript library (not a framework) developed by Facebook for building user interfaces. It is used to build single-page applications where the focus is on creating reusable UI components.

  • Component-Based: You build small, self-contained pieces of code (components) like buttons or headers, and merge them to create complex UIs.
  • Virtual DOM: React creates a lightweight copy of the DOM in memory. When data changes, it updates the Virtual DOM first, compares it to the previous version, and only updates the specific parts of the real DOM that changed. This makes it very fast.

Furthermore, React adopts a declarative approach to coding. In traditional JavaScript, you have to manually tell the browser exactly how to update the page step-by-step (e.g., "find this div, remove this class, add this text"). In React, you simply describe what the UI should look like based on the current data, and React handles the technical details of updating the browser to match that description. This makes your code more predictable and easier to debug. Learning React also opens doors beyond the web; because of its architecture, you can use React Native to build mobile apps for iOS and Android using the exact same concepts.

What you should already know

Before diving into React, you should have a solid grasp of:

  • HTML & CSS: The structure and styling of web pages
  • Modern JavaScript (ES6+): Specifically Arrow Functions, Destructuring, Classes, and Modules.
  • The DOM: Understanding how the Document Object Model works in a standard browser.
React and Angula

Both are powerful tools for frontend development, but they have distinct philosophies:

Feature React Angula
Type Library (Focuses only on the View layer) Framework (A complete solution)
Language JavaScript (JSX) TypeScript
Learning Curve Moderate (Easy to start, harder to master ecosystem) Steep (Many built-in concepts to learn)
Data Binding One-way (Parent to Child) Two-way (Model and View sync automatically)

Code Comparison: The "Counter" Example

1. React Implementation

React uses Functions and Hooks to handle logic. The HTML (JSX) is written directly inside the JavaScript function.

// React: Counter.js
import React, { useState } from 'react';

function Counter() {
  // 1. Define State (Variable)
  const [count, setCount] = useState(0);

  // 2. Define Behavior
  const increment = () => {
    setCount(count + 1);
  };

  // 3. Return View (JSX)
  return (
    

      
Current Count: {count}

      {/* Event binding uses camelCase (onClick) */}
      
    

  );
}

export default Counter;

2. Angula Implementation

Angular uses Classes and Decorators (@Component). It separates the logic (TypeScript) from the view (HTML) more distinctly.

// Angular: counter.component.ts
import { Component } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    

      
Current Count: {{ count }}

      
    

  `
})
export class CounterComponent {
  // 1. Define State (Class Property)
  count: number = 0;

  // 2. Define Behavior (Method)
  increment() {
    this.count++; 
  }
}

Key Differences to Note:

  • State: React uses the useState hook (const [count, setCount]...), whereas Angular simply uses a class property (count = 0).
  • HTML: React puts HTML (JSX) inside the function. Angular usually puts HTML in a separate file (though we used an inline template above for brevity).
  • Clicks: React uses onClick={...} while Angular uses (click)="...".
Hello World

In React, a "Hello World" is usually a Functional Component. This is a JavaScript function that returns HTML (written in a syntax called JSX).


import React from 'react';
import ReactDOM from 'react-dom/client';

// This is a Component
function HelloWorld() {
  return (
    <div>
      <h1>Hello, World</h1>
    </div>
  );
}

// Rendering the component to the HTML file
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render();
Variable

In React, "variables" often act differently depending on whether they are static values or State.

  • Standard Variables: Used for values that don't need to trigger a re-render of the screen (e.g., calculating a math problem inside a function).
  • State Variables (useState): Special React variables that, when changed, tell React to update the screen.
Declaring Variable

Modern React (and JavaScript) moves away from var and uses let and const.


        function UserProfile() {
  // 'let' is used if the value will be reassigned later
  let username = "Guest"; 
  
  // React State (The React way of holding dynamic data)
  const [isActive, setIsActive] = React.useState(true);

  return <h1>Welcome, {username}</h1>
}
Variable scope

Scope determines where your variables can be accessed.

  • Block Scope: Variables declared with let and const inside { curly braces } (like inside an if statement or a loop) only exist inside those braces.
  • Component Scope: Variables defined inside a component function (like UserProfile above) are created every time the component renders and are not accessible outside that component.
Global variables

Avoid using global variables in React. Because React components are meant to be reusable and isolated, relying on global variables can break your app and cause unexpected bugs.

  • Instead of Global Variables: Use Props to pass data down from parent to child, or use Context API / Redux for managing state across the whole application.
Constant

Use const for values that should not be reassigned. In React, you use const for:

  1. Defining Components: const MyComponent = () => { ... }
  2. Importing Libraries: const React = require('react');
  3. State Hooks: The state setter is a constant reference.

const API_URL = "https://api.example.com"; // This value will never change
Data Types

React handles all standard JavaScript data types, but treats them specifically in JSX:

  • String/Number: Rendered directly. <div>{10}</div> displays "10".
  • Boolean/Null/Undefined: valid, but do not render anything on the screen. (Useful for conditional logic).
  • Arrays: React can render lists of items stored in arrays using .map().
  • Objects: You cannot render an object directly (e.g., <div>{userObject}</div> will crash). You must render a specific property: <div>{userObject.name}</div>.
If Else Statement

You cannot use a standard if else statement inside the JSX return block. Instead, you use:

  1. Ternary Operations (inside JSX): 
    
function Greeting({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? <h1>Welcome back!</h1> : <button>Log In</button>}
    </div>
  );
}
  2. Standard if/else (Outside JSX): You can use standard if logic before the return statement to decide what component to render. 
              
function StatusMessage({ status }) {
  if (status === 'success') {
    return <h1 style={{color: 'green'}}>Success!</h1>
  } else {
    return <h1 style={{color: 'red'}}>Error!</h1>
  }
}
